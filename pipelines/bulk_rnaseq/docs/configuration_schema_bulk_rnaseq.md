# Configuration Schema — Bulk RNA-seq
Date: 2026-03-18

## Pipeline
bulk_rnaseq

## Purpose
Define a unified configuration schema for the full active bulk RNA-seq pipeline, including:

- orchestration / wrapper execution
- input discovery and localization
- references and resources
- preprocessing and quantification
- downstream statistical analysis
- reporting and visualization
- logging, metadata, and outputs

This schema is intended to replace the current flat, wrapper-centric config style with a more comprehensive and structured design.

---

## Design Principles

- One schema should cover the full active bulk RNA-seq pipeline
- Wrapper, stage scripts, and downstream analysis should consume the same conceptual parameter model
- Frequently repeated assumptions should be promoted into explicit schema fields
- Stable pipeline conventions may remain fixed, but should still be documented
- The schema should support current bulk RNA-seq needs without over-engineering for future pipelines
- Future extension to scRNA-seq should happen by reusing shared sections, not by weakening the current bulk design

## Section: RUN

### Fields

#### pipeline_name
- Type: string
- Required: yes
- Example: `bulk_rnaseq`
- Notes:
  - Canonical pipeline identifier

#### run_id
- Type: string
- Required: yes
- Example: `crc150_20260318_01`
- Notes:
  - Unique run identifier
  - Should be generated once at wrapper level and propagated downstream
  - Should not be independently regenerated by stage scripts

#### run_mode
- Type: string
- Required: no

## Recommended Fixed Conventions

These do not necessarily need to be freely configurable, but should be explicitly documented as pipeline standards:

- paired-end bulk RNA-seq input
- accepted FASTQ extensions: `.fq.gz`, `.fastq.gz`
- mate suffixes: `_1` / `_2`
- canonical run tree layout
- stage run prefix naming
- tximport expects Salmon per-sample `quant.sf`
- STAR completion markers:
  - `Aligned.sortedByCoord.out.bam`
  - `Log.final.out`
- current GSEA identifier normalization:
  - strip Ensembl version suffix
  - map Ensembl to HGNC symbols for symbol GMTs

---

## Priority Normalization Decisions

These should be addressed during implementation of the schema:

1. Remove duplicate run-id generation across config, wrapper, and stage scripts
2. Replace pilot-oriented variable names such as `PILOT_FASTQ_DIR`
3. Standardize path style across config fields
4. Decide whether sample metadata should continue to be parsed from sample names or come from explicit metadata
5. Decide whether stage-level `run_info.txt` remains local or is unified into centralized metadata
6. Surface dry-run behavior at wrapper level
7. Remove hard-coded `.libPaths()` from tximport stage
8. Replace hard-coded GSEA summary run/date filtering with explicit run selection
9. Convert manually dated downstream output names into run-based standardized names

---

## Implementation Recommendation

For implementation, use this schema conceptually even if the first concrete format remains shell-compatible.

Suggested transition path:

### Phase 1
- Keep shell/env config loading
- Rename variables and group by section in comments
- Expand config coverage to include missing fields

### Phase 2
- Refactor wrapper to consume canonical section-aligned variables
- Pass explicit arguments into stage scripts
- Remove duplicated defaults from stage scripts where possible

### Phase 3
- Add config-driven argument parsing to downstream R scripts
- Standardize run-based output naming
- Unify metadata/logging behavior

### Phase 4
- Consider migration to YAML or TOML if shell env becomes too limiting

## Summary

The unified bulk RNA-seq schema should:

- cover the full active pipeline, not only the wrapper
- separate stable conventions from tunable parameters
- make repeated assumptions explicit
- support current bulk RNA-seq production hardening
- remain extensible for future shared multi-pipeline architecture
