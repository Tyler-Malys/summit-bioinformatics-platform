# Output Documentation

## Purpose

This document describes the structure and contents of pipeline outputs generated by the Summit Informatics v1.0 system.

All outputs are generated within structured run directories under the shared production environment:

    /srv/bioinformatics/pipelines/v1.0

---

## Run Directory Structure

Each pipeline execution creates a run-specific directory:

    runs/<run_id>/

Typical structure:

    <run_directory>/
    ├── input/              # Input files and manifests
    ├── logs/               # Wrapper and stage logs
    │   └── stages/         # Per-stage log files
    ├── outputs/            # Core pipeline outputs (bulk)
    ├── downstream/         # Downstream analysis outputs
    ├── qc/                 # Quality control outputs
    └── run_metadata/       # Execution metadata
        ├── stage_status/   # Stage completion markers
        ├── environment/    # Environment snapshots
        ├── r_sessions/     # R session logs
        └── resolved_config.env

---

## Bulk RNA-seq Outputs

### QC Outputs

Located in:

    qc/

Includes:

- FastQC reports (raw and trimmed)
- MultiQC summary reports
- fastp trimming reports

---

### Core Outputs

Located in:

    outputs/

Includes:

- Salmon quantification results
- STAR alignment outputs (if enabled)
- gene-level counts and TPM matrices

---

### Downstream Outputs

Located in:

    downstream/

Includes:

- DESeq2 differential expression results
- PCA and QC visualizations
- pathway enrichment (FGSEA)
- summary tables

---

## scRNA-seq Outputs

### Preprocessing Outputs

Generated by selected backend:

- Cell Ranger output directories
- STARsolo count matrices

---

### Core Analysis Outputs

Located in:

    downstream/

Includes:

- normalized expression matrices
- dimensionality reduction (PCA, UMAP, t-SNE)
- clustering results
- cluster stability metrics

---

### Downstream Analysis Outputs

Includes:

- marker gene identification
- cell type annotation
- differential expression
- pathway enrichment analysis
- visualization outputs

---

## Visualization Outputs

Located in:

    downstream/analysis/figures/

Includes:

- PCA plots
- UMAP and t-SNE visualizations
- QC plots
- comparison figures

Final compiled reports may include:

- PDF summary reports
- aggregated visualization outputs

---

## Run Metadata

Located in:

    run_metadata/

Includes:

- stage status markers (started, completed, failed)
- environment snapshots (Conda, system variables)
- pipeline version and tool versions
- resolved configuration file
- execution timing and logs

---

## How to Locate Key Outputs

Common locations:

| Output Type | Location |
|------------|---------|
| QC Reports | `qc/` |
| Quantification | `outputs/` |
| Differential Expression | `downstream/` |
| Pathway Analysis | `downstream/` |
| Visualizations | `downstream/analysis/figures/` |
| Metadata | `run_metadata/` |

---

## Interpretation Notes

- QC reports should be reviewed first to assess data quality
- Differential expression results depend on configuration and contrasts
- Pathway results depend on gene set selection and statistical thresholds
- scRNA outputs depend on clustering and normalization parameters

---

## Summary

The output system is designed to provide:

- consistent directory structure across runs
- complete traceability of pipeline execution
- separation of core outputs and downstream analyses
- reproducible and auditable results

Users should rely on the standardized run directory structure when navigating outputs.
